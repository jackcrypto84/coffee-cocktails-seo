# Recipe Image Workflow

This site treats recipe visuals as a required editorial asset, not optional decoration.

## Source of truth
- Live article content: `src/content/articles.ts`
- Live recipe image metadata: `src/content/article-images.json`
- Type wrapper for app imports: `src/content/article-images.ts`
- Prompt and validation logic: `src/lib/recipe-images.ts`
- Hero media component: `src/components/recipe-hero-media.tsx`

## Supported fields
- `imagePath`
- `imagePrompt`
- `imageAlt`
- `imageCaption`
- `visualStatus`
- `glassware`
- `garnish`
- `drinkColor`
- `iceStyle`
- `hotOrIced`
- `foamTexture`
- `milkTexture`
- `vesselType`
- `settingMood`
- `styleTags`
- `layeringNotes`

## Rendering rules
1. Article content loads first.
2. Recipe-image metadata is merged from `src/content/article-images.json`.
3. Missing prompt, alt text, caption, and default image path values are generated by `src/lib/recipe-images.ts`.
4. If a file exists in `public/images/coffee/[slug].jpg` or `public/images/cocktails/[slug].jpg`, the article renders it as the hero image and uses it for Open Graph metadata.
5. If the file does not exist, the article renders a polished fallback placeholder and exposes the visual status and prompt for editorial follow-up.
6. Validation rules flag incomplete or visually weak entries so they can be fixed before promotion.

## Prompt templates
- `prompts/coffee-image-template.md`
- `prompts/cocktail-image-template.md`

## Batch workflow
Generate the 20-coffee and 20-cocktail sample metadata library:

```bash
npm run images:sample
```

Audit the live recipe-image registry and missing-image state:

```bash
npm run images:audit
```

Outputs:
- `content/image-prompts/coffee-image-prompts.json`
- `content/image-prompts/cocktail-image-prompts.json`
- `content/image-prompts/image-prompt-library.md`
- `content/image-prompts/live-image-audit.json`
- `content/image-prompts/live-image-audit.md`

## Validation rules
The system flags entries such as:
- cocktail without glassware
- cocktail without ice style
- cocktail without garnish on garnish-dependent styles
- coffee without hot/iced state or vessel type
- milk-based coffee without milk or foam texture
- layered drinks without layering notes
- prompts that still contain generic fallback language

## Editorial standards
- Every recipe visual must match the actual drink style, vessel, texture, garnish, and service format.
- Dirty lattes, espresso tonics, and other layered drinks need explicit layering notes.
- If the recipe changes meaningfully, update the visual metadata too.
- A page still showing the fallback placeholder should be treated as not fully publish-ready for outreach or promotion.
